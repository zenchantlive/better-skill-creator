# Description Writing

The description in `SKILL.md` frontmatter is primarily about **when to use the skill**.

## Good description characteristics

A strong description:
- starts from trigger conditions or use cases
- mentions concrete task types
- avoids vague hype
- stays readable in one pass

## Weak vs strong

Weak:
- "A powerful framework for world-class skill creation"

Stronger:
- "Use when creating or improving a skill for an AI agent, especially when the skill needs reusable scripts, references, project-local setup, validation, or testable operational guidance."

## Include

- task types
- situations that trigger the skill
- important constraints if they affect applicability

## Avoid

- marketing language
- broad claims with no trigger conditions
- descriptions that explain the full workflow instead of when to use the skill
